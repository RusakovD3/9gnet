# G-Net 9 — Гиперсетевая модель 

## О проекте

G-Net 9 — это исследовательская модель многослойной гиперсети, предназначенная для моделирования, анализа и мониторинга сложных телекоммуникационных систем.

Проект объединяет:

* сетевую топологию;
* многослойную архитектуру;
* тензорное описание состояний;
* SLA/SLO/SLI модели;
* d0sl-политики;
* QoS и очереди;
* мониторинг сети;
* прогнозирование деградации;
* визуализацию гиперсети.

Основная идея проекта — описывать сеть как динамическую гиперсетевую систему, где каждый уровень сети имеет собственную семантику, метрики и тензоры состояния.

# Архитектура уровней

## L0 — Сервисы

Уровень сервисов.

* Voice;
* Video;
* FTP;
* Telemetry.

Для сервисов задаются:

* битрейт;
* latency;
* jitter;
* SLA;
* приоритет.

Каждый сервис подключается к ядру сети.

Используются SLA/SLO модели.

---

## L1 — Абоненты


Реализовано:

* мобильные абоненты;
* проводные абоненты;
* d0sl-политики;
* QoS;
* SLA;
* мониторинг;
* модели очередей.

### Реализованные типы трафика

* Broadcast MP3;
* FTP;
* DNS.

### Поддержка SLA

Используются уровни качества:

| SLA    | Битрейт  |
| ------ | -------- |
| Gold   | 320 kbps |
| Silver | 128 kbps |
| Bronze | 64 kbps  |

Если битрейт падает ниже допустимого значения более чем на 10 секунд — считается, что произошла деградация сети.

Мониторинг выполняется каждые 1 секунду.

# d0sl

Проект использует d0sl-политики.

Реализовано:

* чтение d0sl файла;
* парсинг правил;
* создание SLA профилей;
* применение политик к абонентам.

Пример:

* YouTube SLA;
* VoIP SLA;
* low latency policies;
* bitrate enforcement.

Python-парсер преобразует d0sl в структуры G-Net.

# Вычисления

## Hausdorff distance

Используется для оценки дестабилизации гиперсети.

Позволяет сравнивать:

* baseline состояние;
* деградированное состояние.

---

## Centrality

Используется closeness centrality.

Позволяет определить:

* важность узлов;
* критичность;
* влияние отказа.

---

## Queueing models

Используются модели:

* M/M/1;
* FIFO;
* QoS queues.

Модель учитывает:

* нагрузку;
* задержку;
* деградацию SLA.

---

## Koopman-ready architecture

Архитектура подготовлена для:

* DMD;
* Koopman operator;
* predictive monitoring.
